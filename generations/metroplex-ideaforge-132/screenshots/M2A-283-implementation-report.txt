━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

✓ MCP Protocol Support (JSON-RPC 2.0)
  - initialize: Server handshake with capability negotiation
  - tools/list: List all available skills as MCP tools
  - tools/call: Execute skill functions with parameter validation

✓ Skill Registration
  - Automatic tool registration from installed skills
  - Tool naming: {skill-name}.{tool-name}
  - JSON schema-based input validation

✓ Skill Execution
  - Dynamic module loading with import isolation
  - Timeout handling (default 30s, configurable)
  - Thread-based execution with timeout enforcement
  - Comprehensive error handling and reporting

✓ Execution Logging
  - JSON Lines format (~/.skillhub/logs/execution.jsonl)
  - Tracks: timestamp, skill, tool, parameters, status, duration
  - Filterable by skill name
  - Limit control for query results

✓ Hot-Reload Support
  - Poll-based file modification detection
  - Automatic skill re-registration on changes
  - No external dependencies (no watchdog)

✓ HTTP Server Features
  - Health check endpoint (/health)
  - Graceful shutdown (SIGINT/SIGTERM)
  - Configurable host and port
  - Verbose logging mode

USAGE EXAMPLES
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

1. Start MCP server on default port:
   $ skillhub serve

2. Start with hot-reload for development:
   $ skillhub serve --reload --verbose

3. Start on custom port:
   $ skillhub serve --mcp-port 8080

4. View execution history:
   $ skillhub logs execution-history

5. View last 50 executions:
   $ skillhub logs execution-history --limit 50

6. Filter by specific skill:
   $ skillhub logs execution-history --skill demo-skill

INTEGRATION WITH CLAUDE CODE
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

1. Install a skill:
   $ skillhub install webscraper-skill

2. Start MCP server:
   $ skillhub serve --mcp-port 3000

3. Configure Claude Code to connect to http://localhost:3000

4. Claude Code will:
   - Discover available skills via tools/list
   - Execute skills via tools/call
   - Receive validated results in structured format

5. Monitor execution:
   $ skillhub logs execution-history
